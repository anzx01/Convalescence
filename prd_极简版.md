# 一、产品概述

## 1.1 产品定位

**产品名称**: 安心康复（极简版）
**副标题**: 心脏支架术后康复提醒助手
**产品类型**: 微信小程序（纯前端）

## 1.2 核心功能（仅4个）

1. **用药提醒** - 本地定时提醒（支持早中晚）
2. **打针/复查提醒** - 定期提醒打针和复查
3. **每日打卡** - 记录用药和康复任务
4. **信息推送** - 通过微信消息推送给关注的人

## 3.2 打针/复查提醒

### 功能描述
患者设置打针日期和复查日期，到期前自动提醒

### 提醒类型

#### 1. 打针提醒
- 适用场景：定期打针（如胰岛素、抗凝针等）
- 提醒时间：提前1天、当天早上
- 可设置周期：每周、每月、自定义

#### 2. 复查提醒
- 适用场景：术后复查、定期检查
- 提醒时间：提前7天、3天、1天
- 自动计算：根据手术日期自动生成（1个月、3个月、6个月、12个月）

### 页面设计
```
┌─────────────────────────┐
│  提醒管理                │
├─────────────────────────┤
│                         │
│  📅 复查提醒            │
│                         │
│  下次复查：2026-04-07   │
│  医院：XX医院心内科      │
│  [修改]                 │
│                         │
│  ─────────────────────  │
│                         │
│  💉 打针提醒            │
│                         │
│  下次打针：2026-03-10   │
│  项目：抗凝针           │
│  周期：每周一次         │
│  [修改]                 │
│                         │
│  [+ 添加打针提醒]       │
│                         │
└─────────────────────────┘
```

### 数据存储
```javascript
{
  // 复查提醒
  followups: [
    {
      id: '1',
      date: '2026-04-07',
      hospital: 'XX医院心内科',
      doctor: 'XX医生',
      type: 'surgery_followup', // 术后复查

  // 打针提醒

```

### 提醒逻辑
```javascript
// 复查提醒
function checkFollowupReminder() {
  const today = new Date()
  const followupDate = new Date('2026-04-07')
  const daysUntil = Math.floor((followupDate - today) / (1000 * 60 * 60 * 24))

  if (daysUntil === 7 || daysUntil === 3 || daysUntil === 1) {
    wx.showModal({
      title: '复查提醒',
      content: `还有${daysUntil}天需要复查\n医院：XX医院心内科`,
      confirmText: '知道了'
    })
  }
}

// 打针提醒
function checkInjectionReminder() {
  const today = new Date()
  const injectionDate = new Date('2026-03-10')
  const daysUntil = Math.floor((injectionDate - today) / (1000 * 60 * 60 * 24))

  if (daysUntil === 1 || daysUntil === 0) {
    wx.showModal({
      title: '打针提醒',
      content: daysUntil === 0 ? '今天需要打针' : '明天需要打针',
      confirmText: '知道了'
    })
  }
}
```

## 5.3 提醒实现方案

### 方案1：订阅消息（推荐）
```javascript
// 用户授权订阅
wx.requestSubscribeMessage({
  tmplIds: ['模板ID'],
  success: (res) => {
    // 设置定时任务
    // 到时间后发送订阅消息
  }
})
```

**限制**:
- 需要用户主动授权
- 每次授权只能发送一次
- 需要频繁授权

### 方案2：本地提醒（推荐）
```javascript
// 小程序打开时检查时间
setInterval(() => {
  const now = new Date()
  if (now.getHours() === 8 && now.getMinutes() === 0) {
    wx.showModal({
      title: '用药提醒',
      content: '该服药了'
    })
  }
}, 60000) // 每分钟检查一次
```

**限制**:
- 需要小程序在前台运行
- 用户关闭小程序后无法提醒

### 方案3：混合方案（最佳）
- 小程序打开时：本地提醒
- 小程序关闭时：订阅消息提醒
- 用户每天打开一次小程序即可

# 九、风险与应对

## 9.1 技术风险

| 风险 | 影响 | 应对 |
|------|------|------|
| 提醒不可靠 | 用户忘记吃药/打针 | 引导用户每天打开小程序 |

| 打针日期遗忘 | 错过打针时间 | 提前多天提醒 |
| 复查时间冲突 | 用户记错时间 | 提供日历导出功能 |

## 9.2 业务风险

| 风险 | 影响 | 应对 |
|------|------|------|
| 用户留存低 | 产品失败 | 简化操作，增加打卡激励 |
| 功能太简单 | 用户不满意 | 快速迭代，收集反馈 |
| 无法推广 | 用户增长慢 | 先验证核心价值 |
