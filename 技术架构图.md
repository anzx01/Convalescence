## 2. 数据流向图

```
┌─────────┐
│ 患者操作 │
└────┬────┘
     │
     ↓
┌─────────────────────────────────────┐
│ 1. 小程序发起请求                    │
│    wx.request({                     │
│      url: 'https://api.xxx.com',   │
│      method: 'POST',                │
│      data: {...}                    │
│    })                               │
└────┬────────────────────────────────┘
     │
     ↓
┌─────────────────────────────────────┐
│ 2. Nginx接收请求                     │
│    - SSL解密                         │
│    - 转发到Node.js                   │
└────┬────────────────────────────────┘
     │
     ↓
┌─────────────────────────────────────┐
│ 3. Node.js处理业务逻辑               │
│    - 验证token                       │
│    - 查询Redis缓存                   │
│    - 如无缓存，查询PostgreSQL        │
│    - 处理业务逻辑                    │
│    - 更新数据库                      │
│    - 更新缓存                        │
└────┬────────────────────────────────┘
     │
     ↓
┌─────────────────────────────────────┐
│ 4. 返回响应                          │
│    {                                │
│      code: 200,                     │
│      data: {...}                    │
│    }                                │
└────┬────────────────────────────────┘
     │
     ↓
┌─────────┐
│ 小程序   │
│ 渲染界面 │
└─────────┘
```

## 4. 缓存策略图

```
┌──────────────┐
│  API请求     │
└──────┬───────┘
       │
       ↓
┌──────────────────────────────┐
│  检查Redis缓存                │
│  key: user:{id}:profile      │
└──────┬───────────────────────┘
       │
       ├─── 缓存命中 ──→ 直接返回
       │
       └─── 缓存未命中
              │
              ↓
       ┌──────────────────┐
       │ 查询PostgreSQL   │
       └──────┬───────────┘
              │
              ↓
       ┌──────────────────┐
       │ 写入Redis缓存     │
       │ TTL: 1小时       │
       └──────┬───────────┘
              │
              ↓
       ┌──────────────────┐
       │ 返回数据          │
       └──────────────────┘
```

## 8. 关键技术指标

| 指标 | 配置 | 说明 |
|------|------|------|
| **服务器** | 2核4G | 支持30,000 DAU |
| **数据库** | 1核2G PostgreSQL | 支持50,000用户数据 |
| **缓存** | 1G Redis | 支持80,000会话 |
| **并发** | 1,000-1,500 | 同时在线用户 |
| **QPS** | 300-500 | 每秒请求数 |
| **响应时间** | <200ms | API平均响应 |
| **可用性** | 99.9% | 年度目标 |
